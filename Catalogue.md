## Project Structure Overview

- `crates/` now holds promoted production models: input/core IDs, semantic style, asset slots, observation/decoherence, doors, facility topology, traversal, interaction/equipment, progression/session, networking, and the competitive match brain.
- `labs/` holds independently runnable debug projections and feasibility/integration prototypes. Labs should present or stress a model; durable reusable logic belongs in a shared crate only after reuse is proven.
- `game/` is the assembled player loop that composes the promoted models into menu, lobby, first-person match, HUD, tactical map, audio, loadout, and capture flows.
- `assets/` contains optional drop-in textures, models, and sounds governed by `observed_assets`; procedural/code-as-art presentation remains the default visual strategy.
- `docs/evidence/` contains screenshot evidence for manual verification and visual regression review; regenerate the corresponding image when a visible lab/game behavior changes.
